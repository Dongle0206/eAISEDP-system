# 企业 GRC 方法论（COBIT/COSO/ITIL 裁剪实战版）

## 定位
企业治理/风险/合规三位一体（GRC）。体系"建设和执行"层很完整（战略→架构→流程→项目→数据→产品），本 skill 补"治理监督"层——确保规范被遵循、风险可控、合规达标、持续改进。

## 业界标准依据
- **COBIT 2019**（ISACA）：IT 治理框架——治理目标/管理目标/控制目标
- **COSO ERM**：企业风险管理——内部环境/目标/识别/评估/应对/控制/信息/监控
- **ITIL 4**：IT 服务治理——服务价值系统、四维度模型
- **ISO 31000**：风险管理原则与指南
- **三道防线模型**：业务执行（第一道）/风险合规监督（第二道，GRC 在此）/内审（第三道）

## 六大职能（详见 team-grc 角色定义）
1. 治理策略审视（COBIT）——审视 EA/Standards/Steward 规范是否被遵循、仍适用
2. 治理审计（ITIL/ISAE）——审计流程/数据标准/门禁/控制执行情况
3. 企业风险评估（COSO ERM/ISO 31000）——系统性/合规/战略/数据/供应商风险
4. 合规治理（SOX/COBIT 内控）——内控有效性、法规合规
5. 控制有效性评估（COBIT）——现有控制是否真拦住风险
6. 持续改进驱动（COBIT）——基于发现反哺执行层迭代

## 关键产出规范

### 风险登记册
```markdown
## RISK-{编号}: {风险标题}
- 类别：系统性/合规/战略/数据/供应商
- 描述：xxx
- 可能性：高/中/低
- 影响：高/中/低
- 风险等级：红/黄/绿（矩阵交叉）
- 应对策略：规避/转移/减轻/接受
- 责任人：xxx
- 状态：开放/处置中/已闭环
```

### 治理审计报告
```markdown
# 治理审计报告 — {审计范围/周期}
## 审计对象：xxx
## 审计发现
| 编号 | 严重度 | 发现 | 整改建议 | 责任方 |
## 整改跟踪
## 结论（保留人工签字）
```

## 与执行层的闭环
```
GRC 审计/评估 → 发现缺陷 → 反哺执行层（EA/Standards/Steward/Reviewer）→ 整改 → 复验 → 闭环
```

## 人机边界（硬约束）
AI 做：数据采集、规则检查、风险预警、报告生成、改进建议、控制评估
人工保留：审计结论签字、内控认证、合规处罚、重大风险处置——法定人责
