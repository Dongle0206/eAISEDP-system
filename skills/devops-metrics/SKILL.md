# 交付效能度量方法论（L2 项目群层）

## 触发约定

- `/项目群协同` 流程中定期激活
- PgM 请求效能诊断
- 里程碑复盘

## DORA 四指标体系（业界标准）

Google DORA 研究提出的四大交付效能指标，是衡量软件交付能力的业界共识：

| 指标 | 卓越 | 精英级 | 高 | 中 | 低 |
|---|---|---|---|---|---|
| 部署频率 | 按需多次/天 | 1次/天-周 | 1次/周-月 | 1次/月-半年 |
| 变更前置时间 | <1小时 | <1天 | 1天-1周 | 1周-1月 | >1月 |
| 变更失败率 | <15% | 16-30% | 16-30% | >30% |
| 恢复时间 | <1小时 | <1天 | <1天 | >1天 |

## 流动效率

```
流动效率 = 实际处理时间 / 总前置时间

总前置时间 = 需求登记 → 上线
实际处理时间 = 真正在开发/评审/测试的时间（不含等待）

低流动效率（<15%）= 大量时间浪费在等待，是优化重点
```

## 瓶颈诊断方法

1. **阶段时间分布**：需求/开发/评审/测试/部署各占多少比例
2. **WIP 堆积**：哪个阶段在制品最多
3. **等待类型**：等依赖/等评审/等资源/等部署
4. **横向对比**：各子项目（编排者）效率对比

## 度量→改进闭环

```
度量现状 → 识别瓶颈 → PgM 优化调度 → 复测验证改进 → 沉淀经验
```
