# 组织级知识管理方法论（横向跨层）

## 定位

L1 的经验记忆闭环是**单角色、case 级**的（`docs/agent-memory/dev.md`）。本 skill 把知识沉淀升级到**组织级**——跨项目、跨角色、跨时间复用。

## 四大知识库

| 知识库 | 位置 | 内容 | 维护者 |
|---|---|---|---|
| ADR 决策库 | `docs/knowledge/adr/` | 重大架构决策记录 | EA |
| 复盘库 | `docs/knowledge/postmortems/` | 事故/重大问题复盘（无指责） | SRE + 编排者 |
| 最佳实践库 | `docs/knowledge/best-practices/` | 验证过的模式、方案 | 方法论标准负责人 |
| 模式库 | `docs/knowledge/patterns/` | 可复用的设计/架构模式 | EA + 编排者 |

## 知识流转机制

```
单 case 经验沉淀（L1 agent-memory）
   ↓ 提炼有价值、跨 case 可复用的
组织级知识库（本 skill 管理）
   ↓ 被后续 case/项目检索复用
回灌：编排者派生前检索知识库 → 命中则注入 prompt
```

## 与 L1 经验记忆的区别

| 维度 | L1 agent-memory | 本 skill（组织级） |
|---|---|---|
| 粒度 | 单角色技巧（某 API 怎么用） | 组织决策（为什么选这个架构） |
| 复用范围 | 同角色同项目 | 跨项目跨角色 |
| 维护 | 编排者自动归档 | EA/标准负责人审核入库 |

## 使用场景

- **EA 做架构决策** → 查 ADR 库避免重复决策、查模式库复用方案
- **编排者启动 case** → 检索最佳实践库，把相关条目注入角色 prompt
- **SRE 事故复盘** → 入复盘库，供后续防范
- **方法论标准负责人** → 把最佳实践转成工程标准

## 知识入库标准

不是所有 case 经验都入库——只有满足以下之一才入组织库：
- 跨项目可复用（不是单项目偶发问题）
- 决策级（影响后续选择的架构/选型决策）
- 教训级（踩过的重大坑，值得组织防范）
