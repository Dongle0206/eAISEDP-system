# 工程标准体系方法论（L2 项目群层）

## 触发约定

- `/项目群协同` 流程中激活
- EA 架构原则更新时触发
- 编排者上报规范问题时激活

## 运作模式（标准翻译链）

```
EA 产出架构原则集（高层）
   ↓
方法论标准负责人翻译为工程标准（可执行）
   ├── CLAUDE.md（项目上下文，L1 角色运行时注入）
   ├── 工程标准手册（编码规范/构建规范/部署规范）
   ├── 质量门禁定义（给 Reviewer/Security/QA/Performance）
   ├── 模板库（各角色产出模板）
   └── API 契约规范（给 BA）
   ↓
L1 编排者 + 各角色遵循执行
   ↓
执行中发现标准问题 → 反馈方法论标准负责人 → 必要时反馈 EA 调整原则
```

## 核心产出

| 产出 | 作用 | 使用者 |
|---|---|---|
| CLAUDE.md | 项目上下文注入 | L1 所有角色 |
| 工程标准手册 | 编码/构建/部署规范 | Dev / Reviewer / Ops |
| 质量门禁定义 | 各阶段强制检查项 | Reviewer / Security / QA / Performance |
| 模板库 | 产出格式统一 | PO / BA / QA / Ops |
| API 契约规范 | 接口设计规范 | BA / Dev |

## 标准演进原则

- 标准**随实践演进**——发现遗漏或不可执行就更新，不是一次定死。
- 标准更新要**向下传导**——改了门禁，通知 Reviewer；改了模板，通知各角色。
- 标准**版本化**——重大变更记录变更日志，便于追溯。
