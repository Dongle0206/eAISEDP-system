# 技术雷达方法论（L3 战略层）

## 触发约定

- `/战略投资 <诉求>` —— 显式触发
- 企业规划流程中激活

## 技术雷达四象限

| 象限 | 含义 | 决策建议 |
|---|---|---|
| **Adopt 采纳** | 已验证成熟，推荐广泛使用 | 新项目首选 |
| **Trial 试验** | 有潜力，值得非关键项目试用 | 试点验证后评估升级 |
| **Assess 评估** | 值得关注，需进一步研究 | 持续观察，不投入生产 |
| **Hold 暂缓** | 不推荐新项目使用 | 存量系统可维持，新项目禁止 |

## 维护机制

```
新技术出现 / 现有技术出问题
   ↓
战略分析师 + EA 联合评估（成熟度、风险、社区活跃度、企业适配度）
   ↓
定象限 + 写评估理由
   ↓
更新技术雷达（docs/strategy/tech-radar.md）
   ↓
传导：方法论标准负责人据此更新技术选型约束 → L1 编排者遵循
```

## 与其他层协同

- **输入**：EA 提供技术成熟度评估、性能/安全评估。
- **输出**：方法论标准负责人（L2）把 Adopt/Hold 转成工程标准里的"允许/禁止技术清单"。
- **回灌**：L1 编排者在实际项目中试用 Trial 象限技术的反馈，回灌给战略分析师调整象限。
