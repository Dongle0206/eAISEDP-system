# 供应商/外包管理方法论

## 定位
企业级 IT 必有外包/供应商（乙方开发、SaaS 集成、外包团队）。当前体系假设所有角色都是 AI 内部团队，没考虑供应商。本 skill 补供应商管理维度。

## 核心能力

### 1. 供应商评估与选择
- 能力评估（技术/交付/规模/资质）
- 商务评估（报价/SLA/条款）
- 风险评估（依赖锁定/数据安全/持续经营）
- 评分矩阵与决策

### 2. 合同与 SLA
- 交付范围与验收标准
- SLA 条款（质量/进度/响应）
- 知识产权与数据归属
- 违约条款与退出机制

### 3. 交付验收（关键——供应商代码也过门禁）
- 供应商交付的代码必须过 Reviewer + Security 门禁
- 供应商交付必须过 QA 测试
- 验收清单（功能/性能/安全/文档）
- 不达标打回（与内部 Dev 同等待遇）

### 4. 供应商风险管理
- 依赖锁定风险（核心模块被供应商绑定）
- 交付质量风险（代码质量/进度拖延）
- SLA 违约风险
- 持续经营风险（供应商倒闭/被收购）
- 数据安全风险（供应商接触企业数据）

### 5. 供应商交付标准
由 Standards 定义，供应商必须遵循：
- 编码规范（与内部一致）
- 文档要求（设计/接口/部署/运维）
- 安全标准（与 Security 一致）
- 测试要求（覆盖率/用例质量）

## 与现有角色协同
- GRC 评估供应商风险（企业级）
- Standards 定供应商交付标准
- Reviewer/Security/QA 验收供应商交付（与内部同标准）
- Ops 管供应商交付的部署

## 详细方法论
不新增角色，由 GRC/Standards/Reviewer 调用本 skill。
