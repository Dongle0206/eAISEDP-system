# 产品管理方法论

## 定位
当前 PO 偏"需求拆解员"（出 PRD+AC）。企业级 IT 产品是持续演进资产，需要 Product Manager 视角——管产品全生命周期，不只是单 case。

## 核心能力

### 1. 产品路线图（Roadmap）
- 产品愿景（1-3年）
- 季度主题（每季度聚焦什么）
- 版本规划（Next/Now/Later 框架）
- 与战略对齐（每个版本贡献什么战略价值）

### 2. 需求优先级框架
- **RICE 评分**：Reach(触达)×Impact(影响)×Confidence(信心)÷Effort(投入)
- **Kano 模型**：基本需求/期望需求/兴奋需求
- **MoSCoW**：Must/Should/Could/Won't
- 价值-成本矩阵（速赢/战略投入/暂缓）

### 3. 产品度量（北极星+产品指标体系）
- 北极星指标（产品核心价值单一指标）
- 产品指标树（AARRR 或 HEART）
- 与 Analyst 协同：Analyst 管埋点+分析，PO 管产品决策

### 4. 竞品分析
- 竞品识别与持续跟踪
- 功能差距矩阵
- 差异化策略

### 5. 产品演进
- 技术债与产品债平衡
- 生命周期管理（引入→成长→成熟→衰退）
- 退场策略（功能下线/产品 EOL）

## 与单 case PRD 的关系
- PO 先有**产品路线图视角**（这个产品往哪走）
- 再落到**单 case PRD**（本次需求是什么）
- 路线图指导 case 优先级，case 汇聚成产品演进
