# 数据治理方法论（DAMA 裁剪实战版）

## 定位
企业级数据资产治理。数据治理失败的头号原因是"无主理人"，本 skill 配合 team-steward 提供方法论支撑。

## DAMA 十大职能裁剪到实战

| 职能 | 实战产出 | 主理 |
|---|---|---|
| 数据治理 | 治理章程、数据责任矩阵、认责机制 | Steward |
| 数据架构 | 企业数据模型、数据域、数据流 | EA 定方向，Steward 落地 |
| 数据建模 | 概念/逻辑/物理模型 | DBA 实现，Steward 立标 |
| 数据存储 | 存储选型、性能 | DBA |
| 数据安全 | 隐私、脱敏、访问控制 | Steward 定策略，Security 执行 |
| 数据集成 | ETL、数据流、接口 | Steward 定标准，Dev 实现 |
| 文档/元数据 | 数据字典、血缘 | Steward |
| 主数据/参考数据 | 单一真相源、分发同步 | Steward |
| 数据仓库/BI | 数仓分层、指标体系 | Steward 与 Analyst 协同 |
| 数据质量 | 质量规则、监控、清洗 | Steward |

## 关键产出规范

### 数据标准手册格式
```markdown
# 数据标准 — {实体/字段}
- 业务定义：xxx
- 口径：xxx（含/不含 xxx）
- 数据类型：xxx
- 枚举值：xxx
- 数据质量规则：xxx
- 数据责任：业务责任方/技术责任方
- 关联系统：xxx
```

### 数据目录结构
```markdown
## {数据资产名}
- 描述：xxx
- 责任人：xxx
- 数据域：xxx
- 质量评分：x/5
- 产出系统：xxx
- 消费系统：xxx
- 血缘上游：xxx
- 血缘下游：xxx
- 访问方式：xxx
```

## 价值闭环
治理 → 资产化（可检索可复用）→ 价值挖掘（数据驱动业务）→ 反哺治理（发现质量问题回流）。数据从"负债"变"资产"再变"竞争力"。
